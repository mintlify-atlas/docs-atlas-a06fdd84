---
title: 'Create Announcement'
description: 'Create a new announcement for an event and notify all approved registrants'
---

## Endpoint

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/events/{eventId}/announcements \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "eventId": "clxyz789ghi",
    "title": "Schedule Update",
    "content": "The opening ceremony has been moved to 10 AM"
  }'
```

```typescript TypeScript
const response = await fetch(
  `http://localhost:3000/events/${eventId}/announcements`,
  {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      eventId: 'clxyz789ghi',
      title: 'Schedule Update',
      content: 'The opening ceremony has been moved to 10 AM'
    })
  }
);
```
</CodeGroup>

## Authentication

This endpoint requires authentication. Only event organizers can create announcements.

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

## Path Parameters

<ParamField path="eventId" type="string" required>
  The unique identifier of the event
</ParamField>

## Body Parameters

<ParamField body="eventId" type="string" required>
  The unique identifier of the event (must match the path parameter)
</ParamField>

<ParamField body="title" type="string" required>
  Title of the announcement
</ParamField>

<ParamField body="content" type="string" required>
  Full content/body of the announcement
</ParamField>

## Response

Returns the created announcement object.

<ResponseField name="id" type="string" required>
  Unique identifier for the created announcement
</ResponseField>

<ResponseField name="eventId" type="string" required>
  The event this announcement belongs to
</ResponseField>

<ResponseField name="title" type="string" required>
  Title of the announcement
</ResponseField>

<ResponseField name="content" type="string" required>
  Full content/body of the announcement
</ResponseField>

<ResponseField name="createdAt" type="date" required>
  Timestamp when the announcement was created
</ResponseField>

## Example Request

```json
{
  "eventId": "clxyz789ghi",
  "title": "WiFi Information",
  "content": "The WiFi network is DragonHacks2024. Password: hackathon2024"
}
```

## Example Response

```json
{
  "id": "clxyz123abc",
  "eventId": "clxyz789ghi",
  "title": "WiFi Information",
  "content": "The WiFi network is DragonHacks2024. Password: hackathon2024",
  "createdAt": "2024-03-15T14:30:00.000Z"
}
```

## Notifications

When an announcement is created, all users with **APPROVED** event registrations receive notifications through multiple channels:

### Email Notification
- **Subject**: `[{EventName}] {Title}`
- **Body**: 
  ```text
  New announcement for "{EventName}":

  {Title}

  {Content}
  ```

### WhatsApp Notification
The WhatsApp message uses the following format:
```text
📢 *{EventName}*

*{Title}*
{Content}
```

### Example WhatsApp Message
```text
📢 *DragonHacks 2024*

*WiFi Information*
The WiFi network is DragonHacks2024. Password: hackathon2024
```

## Error Responses

<ResponseField name="FORBIDDEN" type="error">
  Returned when the authenticated user is not an organizer of the event
</ResponseField>

```json
{
  "code": "FORBIDDEN",
  "message": "Only organizers can create announcements"
}
```

<ResponseField name="NOT_FOUND" type="error">
  Returned when the event with the specified ID does not exist
</ResponseField>

```json
{
  "code": "NOT_FOUND"
}
```

## Notes

- Only event organizers can create announcements
- Notifications are automatically sent to all users with APPROVED registration status
- Pending, rejected, and waitlisted registrants do not receive notifications
- The notification process is asynchronous and does not block the API response
- The system logs the number of users notified for debugging purposes
